    bulkGet(opts, cbopt)

    The native bulkGet PouchDB API is not very user friendly. In fact, it's down right wacky! This method patches PouchDB's bulkGet and assumes that all of your requested docs exist. If they do not, it will error via the usual error control flows.

    Source:
    • public.js, line 99
    Parameters:
    Name Type Attributes Description
    opts object | array

    array of {_id, _rev}s, or { docs: [ ... } } where ... is an array of {_id, _rev}s
    cb function <optional>
    Example
    // A good example of what you can expect is actually right out of the tests!
let dummyDocs = [
  { _id: 'a', data: 'a' },
  { _id: 'b', data: 'b' }
]
Promise.resolve()
.then(() => p.save(dummyDocs[0])) // add our first doc to the db
.then((doc) => (dummyDocs[0] = doc)) // update our closure doc it knows the _rev
.then(() => p.save(dummyDocs[1]))
.then((doc) => (dummyDocs[1] = doc))
.then(() => {
  // prepare bulkGet query (set of { _id, _rev}'s are required)
  const toFetch = dummyDocs.map(dummy => ({
    _id: dummy._id,
    _rev: dummy._rev
    // or you can provide .id, .rev
  }))
  p.bulkGet(toFetch)
    .then((docs) => {
      t.deepEqual(docs, dummyDocs, 'bulkGet returns sane results')
      t.end()
    })
})
